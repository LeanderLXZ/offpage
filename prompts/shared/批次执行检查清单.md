# 批次执行检查清单

用途：

- 用于长文本、多阶段、多批次任务的“稀释保护”
- 不要求反复重读整份 prompt，而是在关键时点回看这一份短清单

使用规则：

1. 如果当前任务本来就是 batch 任务，直接按 `batch_id` 使用。
2. 如果当前任务不是 batch 任务，也把当前阶段视为“单批次”。
3. 在以下时点必须回看本清单：
   - 开始第一轮阅读前
   - 每完成一个 batch 或一个大阶段后
   - 准备扩大阅读范围前
   - 准备写文件前
   - 发现自己开始偏离目标时

## 回看清单

1. 当前任务是什么？
   - 只用一句话重述当前目标。
2. 当前处于哪个阶段？
   - 总体分析 / 批次规划 / 候选角色识别 / 世界抽取 / 角色抽取 / 修订合并 / 运行时 / users 回写 / 审校
3. 当前 `batch_id` 或阶段标识是什么？
4. 当前执行模式是什么？
   - 自动连续模式 / 单批手动模式
5. 如果是自动连续模式，当前 tracker 记录的 `next_batch_id` 是什么？
6. 当前允许写入哪些目录？
   - `sources/`
   - `works/`
   - `users/`
   - 或只允许其中一部分
7. 当前明确禁止做什么？
   - 例如：不要通读整本书、不要改写 world canon、不要把用户状态写回 works、不要把小事件写成大事件
8. 当前批次真正要解决的问题是什么？
   - 最多 3 条
9. 当前证据边界是什么？
   - 这批实际允许读取哪些文件 / 章节
10. 当前是否允许修订旧结论？
   - 如果允许，要修订哪些文件
11. 当前输出最少必须包含什么？
12. 你刚刚是否回看了“当前 prompt 的 `稀释保护机制` 段”？
   - 如果没有，先回看，再继续。
13. 如果现在继续往下做，最容易漂移的风险是什么？
   - 先说出来，再继续
