# 批次执行检查清单

用途：

- 用于长文本、多阶段、多批次任务的”稀释保护”
- 不要求反复重读整份 prompt，而是在关键时点回看这一份短清单

使用规则：

1. 如果当前任务本来就是 batch 任务，直接按 `batch_id` 使用。
2. 如果当前任务不是 batch 任务，也把当前阶段视为”单批次”。
3. 在以下时点必须回看本清单：
   - 开始第一轮阅读前
   - 每完成一个 batch 或一个大阶段后
   - 准备扩大阅读范围前
   - 准备写文件前
   - 发现自己开始偏离目标时

## 回看清单

### A. 任务定位（每次回看必做）

1. 当前任务是什么？
   - 只用一句话重述当前目标。
2. 当前处于哪个阶段？
   - 总体分析 / 批次规划 / 候选角色识别 / 世界抽取 / 角色抽取 / 修订合并 / 运行时 / users 回写 / 审校
3. 当前 `batch_id` 或阶段标识是什么？
4. 当前执行模式是什么？
   - 自动连续模式 / 单批手动模式
5. 如果是自动连续模式，当前 tracker 记录的 `next_batch_id` 是什么？

### B. 边界确认（每次回看必做）

6. 当前允许写入哪些目录？
   - `sources/`
   - `works/`
   - `users/`
   - 或只允许其中一部分
7. 当前明确禁止做什么？
   - 例如：不要通读整本书、不要改写 world canon、不要把用户状态写回 works、不要把小事件写成大事件
8. 当前批次真正要解决的问题是什么？
   - 最多 3 条
9. 当前证据边界是什么？
   - 这批实际允许读取哪些文件 / 章节
10. 当前是否允许修订旧结论？
    - 如果允许，要修订哪些文件
11. 当前输出最少必须包含什么？

### C. Schema 与架构规范确认（准备写文件前必做）

12. 如果本轮要写 `stage_snapshots/*.json`，你是否在本 batch 内重读过
    `schemas/stage_snapshot.schema.json`？
    - 如果没有，**现在重读**，再继续写入。
    - 重点确认：voice_state、behavior_state、boundary_state、relationships
      的完整字段列表和必填项。
13. 如果本轮要写 `memory_timeline/*.jsonl`，你是否在本 batch 内重读过
    `schemas/memory_timeline_entry.schema.json`？
    - 重点确认：source_type（canon/inference/ambiguous）是否已标注。
14. 如果本轮要写任何结构化 JSON，你是否在本 batch 内重读过对应的 schema？
    - 不要凭记忆填字段——schema 是权威。
15. 你是否仍然记得以下架构规则？如果不确定，重读
    `simulation/contracts/baseline_merge.md`：
    - 每个 stage_snapshot 是**自包含**的（包含完整状态，不依赖 baseline 合并）
    - 未变化的字段也必须包含在快照中
    - baseline 文件是提取锚点，不在运行时加载
    - voice_rules / behavior_rules / boundaries 是 baseline，不是 override 源

### D. 批次间一致性（batch 2+ 必做）

16. 你是否在本 batch 开始前重读了前一阶段的 stage_snapshot 输出？
    - 用它作为风格和详细度的参照基准。
    - 确认本 batch 产出的快照与前一 batch 在以下方面保持一致：
      - 字段填充详细度（emotional_voice_map 条目数、relationship 条目数等）
      - 术语和命名（角色名、地名、stage_id 格式）
      - evidence_refs 引用密度
      - source_type 标注频率
17. 你是否在本 batch 开始前检查了前一阶段的 memory_timeline？
    - 避免重复记录已有记忆。
    - 确认本 batch 新增记忆的详细度与前一 batch 一致。

### E. 漂移自检（每次回看必做）

18. 你刚刚是否回看了”当前 prompt 的 `稀释保护机制` 段”？
    - 如果没有，先回看，再继续。
19. 如果现在继续往下做，最容易漂移的风险是什么？
    - 先说出来，再继续。
20. 你是否开始出现以下退化信号？
    - stage_snapshot 字段填充变得粗糙（比之前 batch 明显简短）
    - evidence_refs 越来越少或开始省略
    - source_type 全部标成 canon 而没有区分 inference/ambiguous
    - 对话示例（dialogue_examples）越来越少或开始复制前一阶段
    - relationships 条目缺少 driving_events 或 relationship_history_summary
    - 如果出现以上任一信号，**停下来**，重读对应 schema，对照前一 batch
      的输出质量，补齐后再继续
