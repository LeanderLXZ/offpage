# 自动批处理进度模板

用途：

- 让 agent 在“自动连续模式”下，不需要用户逐批指定章节范围，也能持续跑完整个计划
- 用于断点续跑、阶段切换、自动选择下一批

建议用途：

- `works/{work_id}/analysis/incremental/source_batch_progress.md`
- `works/{work_id}/analysis/incremental/world_batch_progress.md`
- `works/{work_id}/analysis/incremental/character_batch_progress/{character_id}.md`

## 建议字段

```text
执行模式：
- 自动连续模式 / 单批手动模式

当前阶段：
- 总体分析 / 批次规划 / 候选角色识别 / 世界抽取 / 角色抽取 / 修订合并

当前对象：
- `work_id`
- 如果适用：`character_id`

最近完成：
- last_completed_batch_id
- last_completed_range
- last_completed_stage_id

下一批：
- next_batch_id
- next_batch_range
- next_batch_goal
- next_stage_id

批次状态：
- pending:
- in_progress:
- completed:
- blocked:
- skipped:

阶段状态：
- 当前阶段是否完成
- 是否允许进入下一阶段

阻塞点：
- 当前需要人工决策的事项
- 当前缺失的数据

备注：
- 自动选择下一批时采用的规则
- `batch_id -> stage_id` 的映射规则
- 当前阶段是否采用“累计到 1..N”的抽取方式
- 本轮修订过哪些旧结论
```

## 自动选择下一批的最小规则

1. 如果存在 `in_progress` 批次，先恢复它。
2. 否则选择当前阶段中第一个 `pending` 批次。
3. 如果当前阶段没有 `pending`，判断是否满足进入下一阶段。
4. 如果进入下一阶段，更新阶段状态与 `next_batch_id`。
5. 只有在下列情况才暂停等待用户：
   - 缺关键输入
   - 规则冲突无法自行决策
   - 需要用户指定目标角色范围
   - 原文证据本身损坏或缺失
