# 批次交接模板

用途：

- 定义每个 batch 结束后需要记录的交接信息字段
- 让下一轮同一个 agent 或新的 agent 不必重新猜当前进度和边界

使用方法：

- 每完成一个 batch，把下面的交接信息更新到对应的 progress 文件中
  （例如 `world_batch_progress.md` 或 `character_batch_progress/{character_id}.md`）
- **不要为每个 batch 单独生成独立的报告文件**
- 如果不是 batch 任务，也可以把当前阶段视为单批次来填写

## 模板

```text
当前阶段：
执行模式：
batch_id / 阶段标识：

本轮实际读取：
- 文件：
- 章节范围：

本轮新增：
- 新增结论：
- 新增文件：

本轮修订：
- 修订了哪些旧结论：
- 改动了哪些文件：

本轮未解决：
- 冲突：
- 不确定点：
- 需要继续核验的地方：

本轮边界提醒：
- 当前禁止做的事：
- 当前仍需遵守的关键规则：

下一步建议：
- next_batch_id：
- 下一批最推荐从哪里继续：
- 下一批开始前应先读哪些文件：
```
