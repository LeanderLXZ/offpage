# 固定 Prompt 库

这个目录存放“可以直接发给一个全新的、没有项目背景的 AI agent”的固定提示词模板。

设计目标：

- 不假设 agent 已经读过本项目文档
- 不假设 agent 已经知道目录结构
- 尽量把关键边界、输出要求、路径规则直接写进 prompt
- 先以可复制的 prompt 模板为主，后续再把真正高频、稳定、带工具约束的流程升级成 skill

使用方式：

1. 选择最接近当前任务的模板文件。
2. 把文件里的“可直接使用的提示词”整段发给新的 agent。
3. 把占位符替换成真实路径、书名、角色名、`user_id`、`context_id` 等。
4. 如有必要，再把少量附加约束补在 prompt 末尾。

建议的使用原则：

- 先用 prompt，把流程和数据边界跑顺。
- 当一个流程“高频、稳定、经常重复、需要固定步骤或脚本”时，再考虑升级成 skill。
- sub-agent 适合做并行工作，不适合代替基础目录结构和基础 prompt 设计。
- 新 agent 不需要默认通读整个 repo，但在写任何目录或结构化文件前，应先回看 [最小结构读取入口](./shared/最小结构读取入口.md)。
- 通过 prompt 启动的新 agent 也不应该完全绕开 `ai_context/`；共享入口现在会先读取最小的 `ai_context` 子集，再进入当前任务。
- `用户入口与上下文装载` 是用户端 runtime 总控，不只是开场加载。
- `users 状态回写` 既可以作为 `用户入口与上下文装载` 的内置持续写回子流程，也可以在需要时被单独调用做补写、修复或 context 合并。

## 稀释保护使用规则

- 不建议在每个 batch 都重读整份长 prompt。
- 更推荐：
  - 总控 prompt 在阶段切换时回看
  - 当前子流程 prompt 在开始本批前回看
  - 每次回看当前 prompt 时，也顺手回看它自己的 `稀释保护机制` 段
  - 每个 batch 在开始前和写回前回看 [批次执行检查清单](./shared/批次执行检查清单.md)
  - 每个 batch 结束后按 [批次交接模板](./shared/批次交接模板.md) 产出交接摘要
  - 用户端多轮会话额外回看 [会话稀释保护检查清单](./runtime/会话稀释保护检查清单.md)
  - 用户端正式写回前额外回看 [写回前防污染检查清单](./runtime/写回前防污染检查清单.md)

## 自动连续模式

- 如果你希望 agent 自动跑完整本书，不应要求你逐批提供章节范围。
- 更推荐：
  - 先生成 `source batch plan`
  - 再建立进度文件，格式可参考 [自动批处理进度模板](./shared/自动批处理进度模板.md)
  - 之后由 agent 自动选择 `next_batch_id` 并持续推进
- 只有在你明确说“单批手动模式”时，才应该由你手动指定本轮 `batch` 范围
