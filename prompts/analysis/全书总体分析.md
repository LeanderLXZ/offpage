# 全书总体分析

适用场景：

- 已经存在 `sources/works/{work_id}/`，需要先做一轮整体分析，但不想一上来就广泛通读整本正文

## 可直接使用的提示词

```text
你现在接手本地项目 `persona-engine`，你没有任何额外背景知识，请完全按本提示词执行。

目标：
基于已有的 source work package，对一本书做一轮“全书总体分析”，为后续世界抽取、角色抽取和用户运行时加载提供方向。

开始实质工作前，先做一次最小结构读取：

1. 先读 `prompts/shared/最小结构读取入口.md`
2. 再读：
   - `README.md`
   - `works/README.md`
3. 如果存在：`works/{work_id}/manifest.json`
4. 如果你仍不确定 `sources / works / users` 的整体关系，再补读 `docs/architecture/data_model.md`

请先从这些文件开始，而不是直接广泛打开正文：

1. `sources/works/{work_id}/manifest.json`
2. `sources/works/{work_id}/metadata/book_metadata.json`
3. `sources/works/{work_id}/metadata/chapter_index.json`
4. 如果已有：`works/{work_id}/analysis/` 下的现有分析文件

分析原则：

1. 不要默认通读整本小说正文。
2. 优先使用：
   - 元数据
   - 章节标题
   - 全书范围的轻量提及统计
   - 少量定点章节核验
3. 明确区分：
   - 原文明示
   - 基于证据的推断
   - 暂不确定
4. 如果这本书体量很大，先建立 source batch plan，再进入深度抽取。

长书处理规则：

1. 如果章节很多、总字数很大、或者显然超过单轮可安全处理的体量，不要试图“一次读完再总结”。
2. 先给出一份分批处理建议：
   - 推荐默认每批 `5` 章
   - 若 `works/{work_id}/manifest.json` 中存在 extraction config，则优先遵循配置
   - 尽量按自然剧情边界切分
3. 总体分析的职责是“确定后续如何分批处理”，而不是取代后续所有批次工作。
4. 默认把 batch `N` 视为阶段 `N` 候选，并在后续抽取中采用“累计到当前阶段、以当前阶段现状为主”的时间模型。

你必须遵守这些项目规则：

1. `sources/works/{work_id}/` 只放原始导入、标准化正文、章节、metadata。
2. `works/{work_id}/` 只放作品级 canon：
   - `world/`
   - `characters/`
   - `analysis/`
   - `indexes/`
3. `users/{user_id}/` 放所有用户侧状态。
4. 中文作品生成的作品相关内容默认用中文。
5. 世界层后续只记录大事件，不记录小桥段。
6. 世界层 cast 只收主角团和高频配角，不收大量龙套。

本轮你要产出的内容：

1. 作品整体结构判断
2. 推荐的 source batch 规划
3. 候选角色列表
4. 主要世界线索
5. 可能的阶段划分线索
6. 推荐的后续抽取顺序
7. 需要继续核验的高风险点
8. 如果后续进入阶段化抽取，当前建议的默认 batch 大小与阶段切分依据

输出要求：

1. 将分析写入 `works/{work_id}/analysis/` 下的合适位置。
2. 内容尽量中文。
3. 不要在回答或日志里粘贴大段原文。
4. 标出你本轮实际打开了哪些章节文件。
5. 如果书很长，必须明确给出“下一批建议从哪里开始”。
6. 最后给出“下一步最推荐做什么”。

稀释保护机制：

1. 开始前先写一张“当前任务卡”：
   - 当前目标
   - 当前阶段
   - 当前允许写入的目录
   - 当前禁止做的事
2. 不需要反复重读整份 prompt，但在以下时点必须回看本 prompt 的这些段落：
   - `目标`
   - `开始实质工作前，先做一次最小结构读取`
   - `分析原则`
   - `长书处理规则`
   - `项目规则`
   - `输出要求`
   - `稀释保护机制`
3. 在以下时点，必须回看 `prompts/shared/批次执行检查清单.md`：
   - 开始 metadata 扫描前
   - 完成章节标题轻扫描后
   - 准备扩大到定点章节核验前
   - 准备落盘前
4. 如果任务还没有进入正式 batch 抽取，也把“当前总体分析阶段”视为单批次执行。
5. 完成这一轮后，按 `prompts/shared/批次交接模板.md` 产出一份简短交接摘要。
6. 一旦发现自己开始滑向“通读整本再总结”，立即停止扩读，先重述任务卡与下一步最小读取范围。

我会补充给你：

- `work_id`：
- 书名：
- 如果已有分析文件，希望你是否参考：
```
