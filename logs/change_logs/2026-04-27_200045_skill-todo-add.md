## 背景 / 触发

会话中讨论时常拍板若干"该做但没立刻做"的事；当前要把它们登记到
`docs/todo_list.md`，每次都要重新对齐 todo_list 文件说明段的字段要求 +
索引段刷新规则，手动跑一遍冗余且容易漏字段或漏刷索引。

用户原始需求："添加一个新的 skill 'todo-add'，把近期在会话中讨论的结果
添加进入 todo_list，默认加入'下一步'，除非指定位置。"

## 结论与决策

新增双文件 skill（按 `/todo` / `/go` 等先例）：

- `.claude/commands/todo-add.md` — slash command 入口
- `.agents/skills/todo-add/SKILL.md` — 同正文 + YAML frontmatter（name /
  description）；正文从一级标题 `# /todo-add` 起与 commands 文件**逐字一致**
- 镜像约束写在文末（与既有 skill 同款约束语）

skill 行为契约（要点）：

1. `$ARGUMENTS` 解析段位：默认"下一步"；`discuss`/`讨论中` →"讨论中（未定案）"；
   `next`/`下一步` →"下一步"；`executing`/`正在执行` →"正在执行"（单槽，需
   显式要求且段为空）
2. 从会话最近若干轮提取要登记的事项；信息不足主动问，不靠猜
3. 合成条目草稿：T-XXX ID（与 todo_list / todo_list_archived 全量去重）+
   字段（按目标段位要求）
4. 给用户预览 → 等确认 → 写入目标段 → 同步刷新顶部索引段（规则**指向**
   `docs/todo_list.md` "如何维护索引"段，不复述）
5. **不 commit / 不 push**——持久化交给 `/commit` 或 `/go`

不修改既有 skill；不动 `docs/todo_list.md` 的"如何维护索引"段（保持那是
唯一权威）。
