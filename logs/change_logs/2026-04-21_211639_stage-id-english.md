# stage_id 统一为紧凑英文代号 `S###`

## 背景

目前 stage_id 使用自由中文字符串（类似 `阶段01_<slug>`），与
`M-S###-##` / `E-S###-##` / `SC-S###-##` / `SN-S###-##` 这套 ID 家族的
stage 段（强制 `S###`）对不齐。prior review 还标出过
`SourceNote.stage_id` 依赖中文匹配会产生风险。

此次把 stage_id 统一为紧凑英文代号 `S###`（三位数字零填充），stage 的
人类可读语义搬到新的 `stage_title` 字段（≤15 字）承载，由 simulation
bootstrap 阶段选择时展示给用户。

磁盘 Phase 3 产物此前已 rollback 至 Phase 2.5 结束状态，
`stage_snapshots/` / `memory_timeline/` 目录均为空，迁移没有 Phase 3
数据破坏面。
