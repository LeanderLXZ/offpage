# Phase 3 重构：后处理程序化 + 并行审校通道 + 提交门控

日期：2026-04-09

## 变更概要

Phase 3 批次提取流程的三项核心重构：

1. **程序化后处理**（0 token）：`memory_digest.jsonl` 和 `stage_catalog.json`
   不再由 LLM 写入，改为提取后由 `post_processing.py` 程序化生成/维护
2. **并行审校通道**：审校从单一 agent 改为 world + 各角色独立通道并行运行，
   每通道各自 validate → review → fix
3. **提交门控**：新增程序化跨通道一致性检查（0 token），全通道 PASS 后才允许
   commit，否则全 batch 回滚

## 新增文件

- `automation/persona_extraction/post_processing.py` — 程序化后处理模块
  - `generate_memory_digest()`: 从 memory_timeline 生成 digest，stage_id 级别幂等
  - `upsert_stage_catalog()`: 从 snapshot 元数据维护 catalog，stage_id 级别幂等
  - `run_batch_post_processing()`: 编排 world catalog + 各角色 digest + catalog
- `automation/persona_extraction/review_lanes.py` — 并行审校通道 + 提交门控
  - `run_parallel_review()`: ThreadPoolExecutor 并行执行各通道
  - `run_commit_gate()`: stage_id 对齐 + snapshot 存在性 + catalog/digest 包含当前阶段
- `automation/prompt_templates/semantic_review_world.md` — 世界层审校模板
- `automation/prompt_templates/semantic_review_character.md` — 角色层审校模板

## 设计决策

- **幂等 upsert**：digest 和 catalog 均以 stage_id 为键做去重追加，安全支持重跑
- **向后兼容**：保留 `EXTRACTED → REVIEWING` 直接转换（旧 progress 文件）、
  保留统一 `semantic_review.md` 模板（兜底）
- **通道写边界不重叠**：world 通道只写 `world/` 下文件，角色通道只写各自
  `characters/{id}/` 下文件，无需加锁
- **snapshot_summary 复用**：`stage_catalog.short_summary` 直接取 snapshot 中
  已有的 `snapshot_summary` 必填字段，无需新增 schema 字段

## Token 节省

- 每个角色提取调用减少 3 个文件输入（baseline_merge.md ~300 行 + digest + catalog）
- 每个世界提取调用减少 1 个文件输入（stage_catalog）
- 审校 reviewer 和定点修复 agent 的输入收窄到通道范围内文件
- chunk summaries 按章节范围过滤，不再传全量 22 个 chunk
