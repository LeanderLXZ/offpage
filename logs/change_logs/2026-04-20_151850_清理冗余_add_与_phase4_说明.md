# 2026-04-20 15:18:50 — git_utils 删冗余 add + extraction_workflow 补 Phase 4 独立说明

## 背景

opus-4-7 audit L1 / L2，纯清理。

- **L1**：`git_utils.commit_stage` 在无 `files` 参数分支里连续调用
  `_git(["add", "works/"], …)` 再 `_git(["add", "-A", "works/"], …)`。
  第二次 `-A` 覆盖第一次，第一次冗余；浪费一个 subprocess，且若有人
  误改第一次的路径，语义会悄悄变化。
- **L2**：`pipeline.json` 中 `phase_4: done, phase_3: pending` 是
  允许行为（两者无时序依赖），但 `extraction_workflow.md` 现有措辞
  "数据独立" 不够显眼，样例 pipeline.json 可能误导新 agent。

## 改了什么

### L1 — 删冗余 add

`automation/persona_extraction/git_utils.py` `commit_stage`：

- 无 `files` 参数分支：去掉 `_git(["add", "works/"], project_root)` 这一行
  与紧跟的独立 `-A` 调用之间的重复；改为分支内直接 `_git(["add", "-A",
  "works/"], project_root)` 一次，`files` 分支保持逐个 add 不变。

### L2 — Phase 4 独立说明上调

`docs/architecture/extraction_workflow.md` §8 场景切分（Phase 4）开头段：

- 新增一句加粗："**Phase 4 可与 Phase 3 并行、先行或后行完成，无时序
  依赖**；`pipeline.json` 中 phase_4 先于 phase_3 完成是允许状态"。
- 其余描述（独立 lock / 独立 start-phase / Phase 3.5 可选门控）不动。

## 校验

- `from automation.persona_extraction import git_utils` import 成功；
  `commit_stage` 签名与 docstring 保留。

## 不做

- `files` 分支保留 "逐个 add" 不变（显式路径语义）。
- `pipeline.json` 样例不改（committed 样例是历史事实）。
