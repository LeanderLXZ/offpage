# Read Scope

## Purpose

This file defines the default reading boundary for future AI agents so they do
not pull unnecessary material into context too early.

Goals:

- save context budget
- reduce noise
- keep attention on what matters for the current task

## Default Priority

Unless the task clearly requires something else, read these first:

- `ai_context/`
- once created and relevant:
  - `works/{work_id}/analysis/`
  - `works/{work_id}/world/`
  - `works/{work_id}/characters/`
  - `users/{user_id}/`
  - specific files under `sources/` that the task directly names

On the first follow-up after handoff, do not proactively read `prompts/`
unless one of these is true:

- the user explicitly asks to use a prompt
- the user names a prompt file
- the current task is prompt design, prompt revision, or prompt execution

## Default Do-Not-Read Areas

Do not proactively scan these areas unless the task directly requires them:

- large raw corpus under `sources/`
- full work-scoped conversation history under
  `users/{user_id}/works/{work_id}/.../sessions/`
- full evidence material under `works/{work_id}/analysis/evidence/`
- full history under `docs/logs/`
- future embeddings, caches, indexes, or intermediate artifact directories
- database files, vector stores, and other large local data
- any large exported or autogenerated artifact trees

## Read These Only When Necessary

Read heavier context only when one of the following is true:

- the user explicitly asks
- the current task depends on specific source evidence
- the compressed AI context is insufficient
- a conflict, inference, or historical revision needs provenance

Examples:

- inspect one chapter excerpt for a line of dialogue
- verify whether a target-character memory point is actually supported by canon
- inspect the source of one relationship change
- inspect one session summary rather than the full transcript
- inspect one specific log entry rather than the full history

## Practical Rule

Prefer:

- targeted reads
- specific files
- minimal necessary excerpts
- summaries first, raw source second

Avoid:

- scanning the full novel by default
- reading the full session history by default
- reading the full log history by default
- loading all evidence files by default
- reading database or index bodies by default
- loading all intermediate artifacts without a direct need
- treating `prompts/` as the default first-stop context source for ordinary
  follow-up work

## Large-File And Source-Handling Rule

- Do not proactively paste full novels, database contents, or long retrieval
  dumps into answers.
- If a task truly depends on source text, read and quote only the smallest
  necessary excerpt.
- If logs or docs need to refer to source material, use summaries and file-path
  references instead of bulk copying.
