# §10 稀释保护范围澄清

**日期**: 2026-04-08
**变更范围**: docs/requirements.md §10

## 背景

§10 原文将"稀释保护"统一描述为提取和运行时两阶段都需要的机制，但
automation 中每个 batch 是独立的 `claude -p` 调用（全新 context），不存在
session 内注意力递减问题。需要区分两种不同风险。

## 修改内容

### §10 引言
- 重写引言段落，明确稀释保护主要针对运行时扮演（§10.1）
- 新增 **适用范围** 说明：automation 中每个 batch 独立，不存在 session 内
  稀释，§10.2 描述的是跨 batch 质量退化

### §10.2
- 标题从"提取阶段的稀释保护"改为"提取阶段的跨批次质量退化防护"
- 移除"指令与规范遗忘"小节（该风险在独立调用架构下不存在）
- 新增 blockquote 注释：说明措施已由 automation 架构天然满足
- 将原来的 5 条编号列表重写为 6 行表格，每条措施映射到 automation 中的
  具体实现（prompt_builder、prompt_templates、validator、consistency_checker）

### 流程图
- 图标题从"提取阶段：批次间的防稀释机制"改为
  "提取阶段：跨批次质量退化防护（§10.2）"
