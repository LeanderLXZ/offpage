# Log And Git Rules Update

Timestamp: 2026-04-01 15:55:31 America/New_York

## Summary

Added project-specific rules for:

- how future AI should use `docs/logs/`
- how major changes should be logged
- how git usage should stay lightweight
- how large local data should be treated

## Main Changes

- added `.gitignore` rules to exclude large local data and runtime artifacts
- added `docs/logs/README.md` to define the purpose and boundaries of logs
- updated `ai_context/instructions.md` with:
  - log update expectations
  - timestamp rules
  - git and repository size rules
  - stronger guidance against uploading large raw data into docs or responses
- updated `ai_context/read_scope.md` to discourage reading large source, log,
  database, and index content by default
- updated `ai_context/README.md`, `ai_context/current_status.md`,
  `ai_context/decisions.md`, and `ai_context/handoff.md` to reflect the new
  policy

## Intent

The project should track code, schemas, templates, and documentation, but it
should not treat raw novel corpus, databases, vector indexes, full user memory,
or large runtime artifacts as normal version-controlled content.

Logs are meant to preserve decision history in compressed form, not to archive
raw source material or bulk generated data.
