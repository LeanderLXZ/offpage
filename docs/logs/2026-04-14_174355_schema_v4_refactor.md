# Schema v4 Refactor — Tier 0 Token Reduction

## 目标

把启动时 Tier 0 加载的 token 从 49 阶段下约 558K 压到约 142K，
同时收紧 ID / 字段语义，为运行时检索器打下可用基础。
本次 refactor 只改 schema / prompt / 校验代码 / 文档；不动运行时加载器
（`simulation/` 下仍是设计文档，无 Python 实现）。

## 核心改动摘要

### 字段重命名

- 全局 `time_in_story` → `time`（scene_archive、memory_timeline、
  world_event_digest、FTS5 过滤字段均统一）
- 角色 snapshot `experienced_events` → `stage_events`
  （语义收紧为"仅本阶段发生"，不再累积）
- 世界 stage snapshot 的 `key_events` **删除**——`stage_events`
  成为世界事件唯一真源，后处理脚本从它生成 `world_event_digest.jsonl`

### ID 规范

统一格式 `{TYPE}-S{stage:03d}-{seq:02d}`：

- `M-S003-07` — memory_timeline 第 3 阶段第 7 条
- `E-S003-07` — world_event_digest 第 3 阶段第 7 条
- `SC-S003-07` — scene_archive 第 3 阶段第 7 个场景

阶段号固定 3 位（最多 999 阶段），序号固定 2 位（单阶段上限 99 条）。
digest 条目**不再携带 `stage_id` 字段**——loader 直接从 ID 前缀用
regex 过滤，节省每条一字段的 token。

### 字段瘦身

- **memory_digest.jsonl**：`{memory_id, summary, importance, time?, location?}`
  - 删除 `stage_id`（从 memory_id 解析）、`emotional_tags`、`involved_targets`
  - `summary` ≤ 50 字；单条目目标 ~30–40 tokens
- **world_event_digest.jsonl**：`{event_id, summary, importance, time?, location?,
  involved_characters?}`
  - 删除 `stage_id`（从 event_id 解析）
  - `summary` ≤ 80 字
- **character stage snapshot → stage_events**：每条 ≤ 80 字
- **memory_timeline**：`event_summary` ≤ 50 字

### 运行时加载策略

- `scene_archive.jsonl` 启动只加载最近
  `scene_fulltext_window`（默认 10）个 **full_text**；
  **summary 不进 Tier 0**，只活在 FTS5 中按需命中
- `identity.json` 走 loader 级字段白名单，启动时剥掉 `evidence_refs`
  （schema 不变；纯加载器行为）
- 方案 A：不改 schema，只改加载

### 数据真源

- `source_batch_plan.json` 是 **stage_id 唯一真源**
- `scene_archive.jsonl` 每次合并由代码全量重建（不手写、不 patch）

### Importance 统一口径

5 级：`trivial / minor / significant / critical / defining`
后处理阶段根据关键词推断 world_event_digest 的 importance。

## 预估收益

49 阶段典型场景下：

| 组件 | 原 | 现 |
|------|-----|-----|
| memory_digest (~60–80 tok/条) | ~60–80 × N | **~30–40 × N**（字段瘦身） |
| world_event_digest | 无 stage 过滤字段压缩 | ID 前缀过滤，条目更轻 |
| scene_archive summary | 全量 | **不加载**（FTS5 兜底） |
| scene_archive full_text | 无上限 | **10 条窗口** |
| identity.json | 全字段 | 白名单（去 evidence_refs） |

合计 Tier 0 约 **558K → 142K tokens**（目标值）。
