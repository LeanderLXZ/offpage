# Phase 0 并行化

**时间**: 2026-04-09 08:34 ET

## 变更内容

将 Phase 0 章节归纳从串行处理改为并行处理，复用 Phase 4 的
`ThreadPoolExecutor` 并行模式。`--concurrency` 参数同时控制 Phase 0
和 Phase 4 的并发数（默认 10）。

## 修改文件

### 核心代码
- `automation/persona_extraction/orchestrator.py`
  - 新增 `_summarize_chunk()` 方法：单 chunk 处理逻辑（LLM 调用 + 写后验证 + JSON 修复）
  - 重构 `run_summarization()`：先过滤已完成 chunk，再用 `ThreadPoolExecutor`
    并行处理 pending chunks，完成后汇总统计
- `automation/persona_extraction/cli.py`
  - `--concurrency` help text 更新为 "Phase 0 and Phase 4"

- `docs/architecture/extraction_workflow.md` — Phase 0 描述更新

- `ai_context/requirements.md` — 更新 §9 和 §11

## 设计说明

- 复用 `--concurrency` 参数，不引入新 CLI flag
- 已完成 chunk 通过文件存在性检测自动跳过（与原有断点恢复逻辑一致）
- 失败的 chunk 不阻塞其他 chunk（串行版会 sys.exit），重跑时自动补全
- 不引入独立的 Phase 0 progress 文件——chunk 输出文件本身即为进度标记
