# /todo-add — 把会话讨论结果加进 todo_list

把当前会话中刚刚讨论 / 拍板的事项写成一条条目，登记到 `docs/todo_list.md`。
默认进"下一步"段；可通过 `$ARGUMENTS` 指定段位。**不 commit**——持久化交给
`/commit` 或 `/go`。

## 步骤

### 1. 解析 $ARGUMENTS（目标段位）

| 取值 | 目标段 |
|---|---|
| 不传 / `next` / `下一步` | `## 下一步` |
| `discuss` / `讨论中` | `## 讨论中（未定案）` |
| `executing` / `正在执行` | `## 正在执行`（单槽，限制见 Step 5） |

非法值 → 打印"段位 `<val>` 未识别，可选: 下一步 / 讨论中 / 正在执行" 并停手。

### 2. 锁定要登记的事项

从当前会话最近若干轮抓取本次要登记的"事项"——通常是用户刚拍板 /
讨论的具体问题 + 结论 + 触发点。

信息不足（缺动机 / 缺现状 / 缺触发链 / 缺改动指向）时**主动问用户补**——
"要登记的是哪段讨论？补一两句关键背景 / 触发点 / 期望结果。" 不靠猜，
不替用户拼接。

### 3. 查 ID 不冲突

`grep -hoE 'T-[A-Z0-9-]+' docs/todo_list.md docs/todo_list_archived.md | sort -u`
取已用 ID 集合。新条目的 T-XXX slug 从内容意图凝练（英文短代号，全大写 +
连字符），与已有集合不冲突。冲突就改名。

### 4. 合成条目草稿

按目标段位的字段要求合成。所有段位共有：

- T-XXX ID + 中文短标题
- **上下文**：动机 + 现状 + 触发链
- **完成标准**
- **依赖**

段位差异：

- **下一步**：必须含 **改动清单**（文件路径 / 行号 / 改动要点），单源不缺
- **讨论中**：必须含 **待决策项**（编号列表，每条 1–2 句）；改动清单可暂缺
- **正在执行**：要求 **开始时间**（YYYY-MM-DD HH:MM EDT）+ **当前状态**
  （进行中 / 等用户决策 / 暂停）

视情况补：**预估** / **未落地原因** / **暂不做的事**。

格式参照 `docs/todo_list.md` 既有条目（"### \[T-XXX\] 中文标题" 起头，
字段标题用 `**字段名**`）。

### 5. 给用户预览 + 等确认

把合成的条目（含 ID + 字段全文）打印给用户，问一句：

```
这样登记到 "<目标段>"？补字段 / 改字段 / 改 ID / 改段位 都行。
```

**不确认前不写文件**。

### 6. 写入 todo_list.md

确认后：

a. 找到目标段（`## 正在执行` / `## 下一步` / `## 讨论中（未定案）`），把
   条目按 `### [T-XXX] 中文标题` 起头追加到该段**末尾**（同段内按用户优先级；
   新条默认尾部，除非用户说"插到最前"）。条目之间用 `---` 分隔（与既有约定一致）

b. **正在执行**段单槽位：写入前 grep 该段已有 `### \[T-` 数；非 0 则**拒绝写入**，
   提示"正在执行段已被占用，请先把当前那条 commit 完成或退回'下一步'再启动新任务"

c. **同步刷新顶部 `## 索引（自动生成，勿手改）` 段**——按 `docs/todo_list.md`
   "## 文件说明 → 如何维护索引"段定义的列规则与字段推断规则刷新对应子表
   一行 + 汇总行；本 skill **不复述规则**，以那段为唯一权威，照规则跑一遍即可

### 7. 收尾报告

打印：

- ✓ 已登记 `T-XXX` 到 "<段位>"
- 索引刷新：`<段>` 行数 X → Y，汇总 N → N+1
- 提示："本 skill 不 commit。要持久化请 /commit 或 /go。"

不进入 /go、不 commit、不 push。

## 约束

- **不 commit / 不 push**
- **不动其他段位**：写入哪段只动哪段 + 索引段，不顺手改别条目
- **ID 查重必须做**：与 `todo_list.md` + `todo_list_archived.md` 全量对照
- **预览必须等确认**：不能合成完直接写盘
- **会话不清晰主动问**：不替用户编字段
- **正在执行单槽**：段非空就拒写
- **索引规则单源**：刷新逻辑指向 `docs/todo_list.md` "如何维护索引"段，
  本 skill 不重复定义；那段改了这里自动跟随
