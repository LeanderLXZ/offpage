# 场景切分任务

你是一个小说文本分析专家。你的任务是将一个章节的原文按**自然场景边界**切分
（通常每章 1-5 个场景，不强求多切），并为每个场景标注元数据。

## 作品信息

- 作品: {work_id}
- 章节: {chapter_id}

## 输入

下面是本章的完整原文（已按行编号）。请仔细阅读全文后再进行切分。

## 切分规则

1. **以自然场景边界切分**：时间跳转、地点转换、视角切换等均为场景边界
2. **不跨章节**：所有场景都在本章内，覆盖全部行
3. **不按固定字数**：场景篇幅由原文自然决定
4. **行号精确**：`scene_start_line` 和 `scene_end_line` 是原文的行号（从 1 开始）
5. **无遗漏无重叠**：所有行必须被某个场景覆盖，相邻场景的行号区间不能重叠
6. **场景数量按实际内容决定**：如果本章自然只有 1 个场景就是 1 个、2 个就是
   2 个——不要为了结构整齐或凑数而切分。**硬约束：每章不超过 5 个场景**；
   如果你数到 6 个以上，说明切得过细，请重新审视相邻场景（小的对话转换、
   情绪起伏、视角微移都不构成场景边界）

## 元数据要求

所有字段均 required；schema 契约 → `schemas/analysis/scene_split.schema.json`，长度上下限以 schema 为准。**bound 是硬上限不是配额**：能 30 字写清的 summary 不要为凑 50 字注水。

- `time`：故事内时间，保留原文的时间表述（可以是模糊的，< 20 字）
- `location`：场景发生的地点（< 20 字）
- `characters_present`：在场的角色列表，使用角色的常用名称
- `summary`：客观第三人称事件梗概，1-3 句话，长度 50-100 字

## 输出格式

请输出一个 JSON 数组，每个元素代表一个场景。**只输出 JSON，不要包含其他文字。**

```json
[
  {
    "scene_start_line": 1,
    "scene_end_line": 45,
    "time": "故事内时间描述",
    "location": "场景地点",
    "characters_present": ["角色A", "角色B"],
    "summary": "客观第三人称事件梗概"
  }
]
```
