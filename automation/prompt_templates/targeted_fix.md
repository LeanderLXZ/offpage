## 修复原则

1. **最小化修改**：只改审校明确指出的字段/条目，不重写整个文件
2. **信息权威层级**（冲突时以高优先级来源为准）：
   - 原文章节 > 章节摘要 > memory_timeline > stage_snapshot > world_snapshot
   - 多个来源一致 > 单个来源孤立说法
3. **跨文件一致性**：如果同一事实出现在多个文件中（如 memory_timeline 和
   stage_snapshot 都记录了某事件），确保修复后所有相关位置一致
4. **不添加新内容**：不要"顺便"补充审校未提到的内容
5. **保持 JSON 格式**：修改后确保文件仍然是合法 JSON/JSONL

## 执行步骤

1. 逐个读取上面列出的"有问题的产出文件"
2. 对照"审校发现的问题"和"原文证据"，定位需要修改的具体位置
3. 使用 Edit 工具做最小化修改
4. 修改完成后，对每个修改过的文件做一次完整性自查：
   - JSON 语法是否正确？
   - 修改是否引入了新的不一致？
   - 是否有其他文件引用了同一事实需要同步修改？
