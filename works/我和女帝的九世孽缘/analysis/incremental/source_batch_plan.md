## 总处理策略

1. 全书按连续 `10` 章切分，最后一批为 `7` 章。
2. 默认把 `batch_N` 作为 `阶段N` 候选，世界抽取时累计处理 `1..N`。
3. 每批优先解决共享世界问题：
   - 阶段世界状态
   - 大事件与历史节点
   - 地点、势力、共同危机
   - 主角团与高频配角在世界层的索引
4. 角色详细心理、口癖、记忆解释不在本计划里落定，只在 world 层记录必要的共享事件和关系信号。
5. 后续 agent 在自动连续模式下应：
   - 先读本文件
   - 再读 `world_batch_progress.md`
   - 优先恢复 `in_progress`
   - 否则选择第一个 `pending` 的 batch

## 推荐优先顺序

1. `batch_001`：先建立首个 world 包、阶段目录与阶段快照。
2. `batch_002`：确认柳村/南林危机的收束方式，判断阶段 2 的世界现状是否发生稳定转场。
3. `batch_003`：继续补齐南林之外的行动空间、哈弟与姜寒汐底牌等新世界线索。
4. `batch_005` 与 `batch_006`：尽早核定萧浩、楚妍儿等高频支线角色进入共享世界层的时点。
5. `batch_012` 之后：随着中州、圣地、仙界、蓝星等标题信号出现，世界层的地点 / 势力 / 地图 / 历史尺度会明显扩张，应保持每批都更新 `world/stage_catalog.json` 与对应快照。
